# 銀行明細 月次シート自動生成ツール

毎月の「銀行明細を6枚のスプレッドシートに転記して『会計データ ※月ごと』の月次フォルダに置く」作業を自動化する Google Apps Script (GAS) ツールです。

## 何が自動になるか

従来の手作業（毎月30〜60分）:

1. 各ネットバンクで照会画面を開く
2. **新規スプレッドシートを6枚作成**（【SP】みずほ銀行明細 / 月末残高、【EM】みずほ銀行明細 / 月末残高、【西武信用金庫】明細 / 月末残高）
3. 照会画面をそれぞれ貼り付け
4. **月次フォルダ（例: 202606）を作成して配置**
5. **SP明細に 販管費 / 人件費 / 税金 / 社保 の分類集計と合計を手計算で追記**

このツール導入後は、**貼り付け（手順3相当）だけが手作業**で、2・4・5 が自動になります。銀行サイトへのログインだけは二段階認証があるため自動化していません（安全のため）。

- ファイル名・シート形式は従来と同一（税理士さん側の見え方は変わりません）
- 分類集計は摘要のキーワード一致で自動計算（ルールは設定タブで編集可能）
- 出金合計・入金合計を自動追記、生成結果ダイアログで検算値を確認できる
- 同名ファイルが既にある月はスキップ（上書き事故防止）

## 初回セットアップ（5分・1回だけ）

1. Google ドライブで新規スプレッドシートを作成し、名前を「銀行明細_自動転記」などにする
2. メニュー「拡張機能」→「Apps Script」を開く
3. エディタの `コード.gs` の中身を全部消し、`Code.gs` の内容を貼り付けて保存（Ctrl+S）
4. スプレッドシートに戻ってリロード（F5）
5. 新しいメニュー「🏦 銀行明細ツール」→「① 初期セットアップ（タブ作成）」を実行
   - 初回は Google の承認画面が出るので、自分のアカウントで許可する
6. 「設定」タブを確認:
   - `出力先親フォルダID` … 「会計データ ※月ごと」フォルダのIDが初期設定済み（URLの `folders/` 以降の文字列）
   - `人件費とみなす最低金額` … 初期値 100,000円。給与振込と少額の立替精算を区別するための閾値
   - `分類カテゴリ` … 摘要キーワードのルール。**人件費の氏名リスト（半角カナ）は実際の給与振込先に合わせて必ず見直してください**

## 毎月の使い方（5分）

1. 各ネットバンクで対象月の「入出金明細照会」「残高照会」を表示
2. 画面をコピーし、対応する貼り付けタブ（`SPみずほ明細` など6枚）へ A1 起点で貼り付け
   - 従来「新規シートに貼り付け」していたのと同じ操作です
3. メニュー「🏦 銀行明細ツール」→「② 月次シート生成」
   - 貼り付け内容の日付から対象月を自動判定し、確認ダイアログ後に生成します
4. 結果ダイアログで件数・出金入金合計・分類額を確認
   - 分類はキーワードによる自動判定なので、SP明細の分類額だけ目視確認を推奨
5. 「③ 貼り付けタブをクリア」で次月に備える

一部の口座だけ貼り付けて実行してもOKです（空のタブはスキップされます）。

## カスタマイズ

- 口座の追加・ファイル名変更: 「設定」タブの出力定義（貼り付けタブ名 / 出力ファイル名 / 種別 / 分類列を付ける）を編集
  - 行を増やす場合は `Code.gs` の `CFG_OUTPUT_DEF_COUNT` も合わせて変更
- 分類ルールの追加: 設定タブの分類カテゴリ表にキーワードを追記（カンマ区切り、摘要の部分一致）

## 制限・注意

- 銀行サイトからの明細取得（ログイン）は自動化対象外。コピペ1回/口座は残ります
- 分類集計はあくまで下書きです。月次の締めでは必ず目視確認してください
