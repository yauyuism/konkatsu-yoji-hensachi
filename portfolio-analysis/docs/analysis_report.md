## 2. 問題点

サンプルPFから読み取れる典型的な論点（あなたの実データでも同種の観点でチェックします）。

1. **地域が北米に84%集中**。「全世界株」を持っていても、S&P500・NASDAQ100・SCHDが全て米国のため、
   実質は米国一本足に近い。米国が強い前提に賭けた状態で、地域分散は効いていない。
2. **通貨がUSDに86%集中**。円建て生活者にとって、為替（円高）で評価額が目減りするリスクが大きい。
3. **実質的な重複投資**。S&P500・全世界株・NASDAQ100・SCHDは中身（Apple/Microsoft/NVIDIA等の大型テック）が
   大きく重なる。銘柄数は多いが、リスクの源泉は「米国大型テック」に一極化。
4. **NASDAQ100が21%とやや過大**。単独ボラ25%と高く、下落局面での振れ幅を増やしている。
   しかも信託報酬0.2035%はNASDAQ100の中でも最安ではない（→ コスト最適化余地）。
5. **債券・現金・金などの"守り"がゼロ**。想定最大DDが−55%目安。暴落時に買い増す原資（待機資金）が
   ポートフォリオ内に無く、下落に対して受け身。
6. **NISA枠を使い切れていない**。年間の特定口座積立24万円がある一方、つみたて枠に60万円・成長枠に192万円の
   空きがある。非課税で運用できる分を課税口座で運用している（税効率のロス）。

## 3. 改善案

**方針**: 「増やす力（期待リターン）」は活かしつつ、**集中の是正**と**税効率の改善**、そして
**下落を利用する仕組み**（待機資金）を足す。具体的には以下。

- **A. 集中の是正**: 米国＋ハイテクの比率を、全世界株（オルカン）というコアに寄せて相対的に下げる。
  「米国が強い」見立てを否定するのではなく、外れたときのダメージを許容範囲に収める。
- **B. 通貨分散**: 為替ヘッジ債券・日本資産・金を一部加え、USD一極を緩和。
- **C. 重複の整理**: NASDAQ100はサテライト上限（例: 15%以下）を決める。高配当は目的（配当か値上がりか）を明確化。
- **D. 税効率**: コア（オルカン/S&P500）をNISA優先へ。空いている非課税枠（つみたて60万・成長192万）を先に埋める。
- **E. 守りと待機資金**: 債券・現金・金でクッションを作り、暴落時に買い増す原資を PF 内に確保（→ 運用ルール§2）。

改善は**一括で組み替えず、新規積立の配分変更で時間をかけて寄せる**（ノーセル・リバランス）。
課税口座の含み益を売ると税金が発生するため、まず「これから入れるお金の向き先」を変えるのが基本。

## 4. 推奨ポートフォリオ（STEP3・3パターン）

`portfolio_lab/config.py` の `MODEL_PORTFOLIOS` に定義。数値はツールが分散共分散法で算出。

### 守備型（最大ドローダウンを抑える）
- 期待リターン **3.3%** / 想定ボラ **6.7%** / 想定最大DD **−19%(目安)**
- 全世界株25 / 高配当10 / 先進国債券(ヘッジ)30 / 日本債券10 / 金10 / 現金15
- 想定読者: near-retirement、下落に強いストレス、5年以内に使う予定がある資金。

### バランス型（リスクとリターンの均衡）
- 期待リターン **4.8%** / 想定ボラ **11.6%** / 想定最大DD **−36%(目安)**
- 全世界株50 / 新興国5 / 高配当10 / 先進国債券(ヘッジ)20 / 金5 / 現金10
- 想定読者: 10〜20年の資産形成で、下落もある程度受け入れつつ夜眠れるバランスを求める人。

### 攻撃型（長期で最大リターンを狙う）
- 期待リターン **6.3%** / 想定ボラ **18.0%** / 想定最大DD **−52%(目安)**
- 全世界株50 / 米国株20 / 米国ハイテク15 / 新興国10 / 現金5
- 想定読者: 20年以上の長期、暴落でも積立を止めない胆力、当面使う予定のない資金。

> サンプルPF（想定リターン6.7% / ボラ19.1% / DD−55%目安）は「攻撃型よりさらに攻めた」水準。
> 同じ攻めるでも、全世界株コア化で**ボラとDDを下げつつ期待リターンはほぼ維持**できる、というのが3モデル比較の含意。

各モデルへ寄せるためのリバランス提案（増やす/減らす/許容帯・調整額）は
`reports/rebalance_plan.csv` に自動出力されます。

## 5. 実装コード（STEP4 自動分析環境）

`portfolio-analysis/` 一式が実装です。設計の要点:

- **データ取得/管理**: `data/*.csv`（保有・投信参照・資産クラス前提）を `data_sources.py` が読み込み、
  銘柄名から資産クラス・信託報酬・ベンチマークを自動解決（部分一致対応）。
- **分析エンジン**: `analytics.py`
  - 加重信託報酬、資産クラス/地域/通貨のルックスルー配分、重複検出、
  - 分散共分散法での想定リターン・ボラ、株式比率からの最大DD目安。
- **リバランス/NISA**: `rebalance.py` … 目標比率との乖離→売買額、NISA枠の空き→優先配分アドバイス。
- **レポート**: `reports.py` … Markdown月次レポート・CSV3種・資産推移の投影、
  matplotlibがあれば配分円グラフ・推移グラフ・PDFを生成。
- **出力チャネル**: CLI（`cli.py`）と Streamlitダッシュボード（`app/streamlit_app.py`）。

実行:
```bash
python -m portfolio_lab.cli --holdings data/holdings.csv --target balanced --out reports
streamlit run app/streamlit_app.py
```
前提値（`data/asset_class_assumptions.csv`・`config.py`）を変えれば結論も変わります。
**"数値の出所"を差し替え可能にしてある**のがこのツールの肝です。

## 6. 運用ルール（STEP5）

詳細は [`operation_rules.md`](./operation_rules.md)。骨子:

- **積立**: 先取り自動積立、NISA優先、コア(全世界/S&P500)＋サテライト上限。金額は固定。
- **暴落時**: 生活防衛資金とは別の待機資金を、−20/−30/−40%で3分割投入（価格トリガーのみで判断）。
- **利確**: 資産形成期は原則売らない。売るのは①ライフイベント②リバランス③出口フェーズの取り崩しのみ。
- **リバランス**: 年1回＋±5%閾値、まず新規資金で直す（ノーセル）。
- **NISA**: つみたて→成長→特定の順。コアを非課税枠へ。
- **感情対策**: 売買前チェックリストで「ルール由来か気分由来か」を必ず確認。迷ったら何もしない。

## 付録: 経営者のキャッシュアロケーション視点

> 会社経営者の場合、最適化すべきは「投資信託の中の配分」ではなく **総資産（個人＋法人＋事業）の配分** です。
> ここは一般論としての意思決定フレームで、税務・会計の具体は顧問税理士の確認前提です。

### A. 4つのバケツで考える

| バケツ | 期待リターンの目安 | 流動性 | 役割 |
| --- | --- | --- | --- |
| ① 本業への再投資 | 事業ROI（多くは10〜30%+） | 低 | 最も高リターンになり得る。ただし集中リスク・自分の時間依存 |
| ② 事業の運転/防衛現金 | ほぼ0% | 高 | 数か月〜1年の固定費。これがないと①③を守れない |
| ③ 金融資産（投信等） | 4〜7%（本書の前提） | 中〜高 | 事業と相関の低い分散資産。ここが個人PFの領域 |
| ④ 法人内部留保・現金比率 | 0〜数% | 高 | 与信・投資機会・不況耐性のバッファ |

### B. 意思決定の原則: 限界リターンで比べる

- **追加の1円は、①〜④で"次の1円の期待リターン(リスク調整後)"が最も高い所へ。**
  事業ROIが金融資産の期待リターン(4〜7%)を**継続的に**上回るうちは、本業再投資が合理的。
- ただし①は**集中・非分散・自分の労働に依存**。事業ROIには「そのリスクに見合う上乗せ(リスクプレミアム)」を
  差し引いて、③の分散資産と比較する（例: 事業の"リスク調整後ROI"がまだ7%超なら①優先、下回り始めたら③へ配分）。
- **相関で守る**: 事業キャッシュフローが景気連動なら、③はディフェンシブ寄り（債券・金・全世界分散）にして
  「本業が不調な局面で金融資産も一緒に沈む」共倒れを避ける。事業がディフェンシブなら③は攻めてよい。

### C. 個人 vs 法人（器の選択）

- **法人での運用**: 損益通算・経費・退職金設計などの自由度がある一方、法人税・出口設計が絡む。
- **個人での運用**: NISA(非課税)・iDeCo(所得控除)など**制度上の"確実なリターン"**が使える。
  税制優遇枠は「リスクゼロで数十%のリターン」に相当し得るため、**個人の非課税枠は最優先で埋める**のが定石。
- 一般的な優先順位の一例: 事業の高ROI再投資 → 個人NISA/iDeCo枠 → 法人/個人の課税運用 → 内部留保積み増し。
  （事業ROIと各手段の税引後リターンを、あなたの実数で比較して決める。）

### D. このツールでの拡張

`data/holdings.csv` に法人保有分や現金を1行ずつ（`account_type` を `corp` 等に）加えれば、
③④を含めた総資産の配分・通貨・分散を同じ枠組みで可視化できます。①本業ROIと③期待リターンの比較は、
`docs/interview_template.md` の「経営者向け追加ヒアリング」に実数を入れて判断してください。
